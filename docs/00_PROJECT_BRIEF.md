# 00 — Project Brief

## Name

Energy Flex Mini-Lakehouse

## Learning target

Review, refresh, consolidate, and refine data engineering skills, thinking, and tooling through a coherent end-to-end capstone.

## Why this project exists

The goal is not broad tool familiarity. The goal is to build stronger engineering judgment through a compact, coherent capstone.

This project should train and exhibit:

- correctness under temporal ambiguity;
- production-grade Python discipline;
- SQL and data modeling depth;
- Spark/distributed-processing reasoning;
- streaming semantics;
- operational maturity;
- clear senior-level communication.

## Engineering orientation

The project emphasizes:

- correctness;
- speed;
- reliability;
- observability;
- scalability;
- architectural judgment;
- autonomy;
- pragmatic scientific thinking.

The project focuses on data infrastructure with operational and analytical consequence.

## Core project principle

Every artifact should strengthen practical data-engineering capability through implementation, reasoning, and repeatable workflows.

If an implementation detail does not support the learning objective, simplify it.
